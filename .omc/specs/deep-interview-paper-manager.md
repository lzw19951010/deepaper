## Goal
构建一个个人论文管理工具（Python CLI），核心流程为：输入arxiv链接 → 通过HuggingFace Hub下载PDF → 使用Claude Opus API按固定字段prompt模板解析论文 → 输出结构化Markdown笔记（含YAML frontmatter） → 自动打标签分类 → git同步到GitHub私有仓库 → 本地用Obsidian浏览和查询。支持基于标签和解析内容的检索（ChromaDB向量搜索 + Obsidian Dataview），支持RAG语义检索，让用户在想到某个idea但忘记论文名时能快速找到相关论文。

## Core Pipeline
```
arxiv链接
  → HuggingFace Hub 下载PDF（中间产物，不保留）
  → Claude Opus API 解析（用户自定义固定字段prompt模板）
  → 结构化Markdown + YAML frontmatter（标题、作者、关键词、标签等）
  → 自动标签分类（Claude生成）
  → 本地存储为 .md 文件
  → git push 到 GitHub 私有仓库（多端同步）
  → Obsidian 打开仓库目录作为 vault（浏览/查询/Dataview）
  → ChromaDB 向量索引（RAG语义检索）
```

## Constraints
- **编程语言**: Python（生态最适合：huggingface_hub, anthropic, chromadb, gitpython）
- **LLM**: Claude Opus（Anthropic API），用于论文解析和自动标签
- **论文获取**: HuggingFace Hub 下载 arxiv 论文 PDF
- **输出格式**: Markdown文件 + YAML frontmatter，固定字段模板（参考Zotero常用字段）
- **Prompt模板**: 用户后续提供具体模板内容，系统支持自定义prompt
- **存储**: 本地Markdown文件 + git版本控制（push到GitHub私有仓库）
- **同步方案**: Git同步（GitHub免费版，10,000篇论文约30-100MB，远低于1GB限制）
- **本地浏览**: Obsidian打开git仓库目录作为vault
- **RAG检索**: ChromaDB本地向量数据库 + embedding模型
- **跨平台**: Windows和Mac（git + Obsidian + Python均跨平台）
- **不存储PDF原文**: PDF仅作为中间产物，解析后不保留
- **检索范围**: 标签检索 + 解析内容语义检索（不做原文全文检索）
- **容量估算**: 10,000篇 × 3-5KB/篇 = 30-50MB（git完全承受）

## Non-Goals
- 不是论文阅读器（不提供PDF阅读功能）
- 不做原文全文检索（成本太高）
- 不做可视化知识图谱（第一版）
- Zotero集成不是核心功能（nice-to-have，用户之前用Zotero但不强制要求）
- 不做手机端（第一版）
- 不做多用户/协作功能
- 不做Web UI（第一版用CLI + Obsidian）
- 不用Notion（选择了git + Obsidian方案）
